Objetivo:
Desarrollar una plataforma de monitoreo energético para un usuario, que utilice datos de consumo
energético para tres centros de trabajo, distribuidos en diferentes áreas y con varios sensores por centro.
La plataforma deberá mostrar, en tiempo real, el voltaje y la corriente medida por cada sensor,
permitiendo al usuarios visualizar y gestionar los datos de manera eficiente. El sistema debe incluir
autenticación de usuario y ser desplegado en la nube.
Datos:
Se te proporcionará un archivo JSON con datos simulados de consumo energético para tres centros de
trabajo. Cada centro de trabajo tiene sensores distribuidos en diferentes áreas y las mediciones incluyen
voltaje y corriente. Las lecturas se realizan cada 15 minutos durante una semana.
Formato del archivo JSON:
Centro de trabajo: Nombre del centro de trabajo (Centro 1, Centro 2, Centro 3).
Sensor ID: Identificador del sensor dentro de cada centro.
Área: Área dentro del centro de trabajo donde se encuentra el sensor.
Fecha: Fecha de la medición.
Hora: Hora de la medición.
Voltaje (V): Medición de voltaje.
Corriente (A): Medición de corriente.
Requisitos Técnicos:
Utilizar Node.js para el backend y React para el frontend.
Crear una API que gestione las mediciones de voltaje y corriente contenidas en el archivo JSON
proporcionado.
La aplicación debe permitir que el usuario pueda acceder y ver los datos divididos por centro de
trabajo.
Implementar un filtro que permita a los usuarios consultar los datos por fecha, sensor y área.
Backend:
La API debe recibir y gestionar los datos del archivo JSON, simulando que estos provienen de los
sensores IoT.
Permitir que el usuario realicen consultas para obtener el historial de consumo de sus sensores, así
como las lecturas en tiempo real (extraídas del JSON).
Incluir una funcionalidad para calcular estadísticas simples, como el promedio de voltaje y corriente
por área o sensor.
Frontend:
Crear un dashboard en React que permita a los usuarios visualizar:
Las mediciones de voltaje y corriente de sus sensores en tiempo real.
Gráficos interactivos que representen las mediciones históricas.
Filtros para visualizar los datos por área, sensor y fecha.
Despliegue en la nube:
Desplegar la API y el frontend, asegurando que ambas partes de la aplicación estén correctamente
integradas y accesibles.
Utilizar MongoDB Atlas o Azure CosmosDB para almacenar y gestionar los datos si decides no trabajar
directamente con el JSON estático.
Entregables:
Código Fuente:
El código fuente completo de la aplicación, debidamente estructurado y comentado.
Incluir un archivo README con instrucciones claras sobre cómo ejecutar la aplicación localmente y
cómo acceder a la aplicación desplegada.
URL de Despliegue:
Proporcionar la URL de la aplicación desplegada, tanto para el backend como para el frontend.
Documentación Técnica:
Explicar brevemente la arquitectura utilizada y las decisiones técnicas importantes.
Describir cómo se gestionan los datos, especialmente en relación con la autenticación de usuario y la
segregación de datos entre los distintos centros de trabajo
Plazo para la Entrega:
El candidato tendrá un plazo de 3 días para completar la prueba técnica, incluyendo el despliegue y la
entrega de todos los componentes requeridos.
Criterios de Evaluación:
Funcionalidad y Completitud:
El sistema debe ser completamente funcional, con la capacidad de gestionar el acceso del usuario
y mostrar las mediciones de los sensores del archivo JSON.
Calidad del Código:
Se evaluará la estructura, claridad y organización del código.
Se valorará el uso de buenas prácticas de desarrollo (manejo de errores, seguridad de la API,
etc.).
Frontend:
La interfaz debe ser clara, accesible y permitir una navegación fluida para el usuario.
El uso de gráficos para mostrar los datos energéticos será un punto importante.
Despliegue en la Nube:
Se evaluará la correcta configuración del despliegue, asegurando que el backend y frontend estén
correctamente enlazados y accesibles.
Documentación:
La documentación debe ser clara y precisa, facilitando la comprensión del proyecto y los pasos a
seguir para su despliegue o ejecución.
